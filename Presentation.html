It's Not Compiling, It's "Freezing"

Your `.py` script, all its libraries, and the Python interpreter itself are **bundled** into a self-extracting package.

The goal is **distribution**, not performance.

Pros vs. Cons

PROS ‚úÖ

  • Accessible: Anyone can run it.
  • Reliable: Bundled dependencies prevent conflicts.
  • Protected: Hides your source code.

CONS ‚ùå

  • Huge Files: Includes the entire interpreter.
  • Slow Startup: Must unpack itself first.
  • Platform-Specific: Build on Windows for Windows, etc.
  • Antivirus Issues: Often gets false-flagged.

Live Demo Time!

Let's build a "Quote of the Day" app.

# 1. Create & activate venv

> python -m venv venv

> source venv/bin/activate


# 2. Install packages

(venv) > pip install pyinstaller requests

The Build Command

> pyinstaller --onefile --windowed --icon="logo.ico" --add-data="logo.ico:." quote_app.py

--onefile

Single .exe

--windowed

No console

--icon

Set file icon

--add-data

Bundle assets


Check the "dist" folder in your venv for your finished file.

Pro Tips & Troubleshooting

  • File Size Too Big? Use a free tool called UPX to compress your EXE.
  • Startup Too Slow? Distribute the faster "one-dir" folder instead.
  • Antivirus Warning? Use VirusTotal.com to diagnose and report the false positive.
